# Valid Anagram

## Условие

Две строки называются **анаграммами** друг друга, если одна из них может быть преобразована в другую перестановкой её символов.

Дано две строки `s` и `t`. Определить, является ли одна из них анаграммой другой.

**Требования:**
- Проверить, содержат ли строки одинаковые символы в одинаковых количествах
- Учитывать регистр символов (прописные и строчные разные)
- Поддерживать Unicode символы

## Примеры

Input: `s = "anagram"`, `t = "nagaram"`  
Output: `true`  
Объяснение: Обе строки содержат a(3), n(1), g(1), r(1), m(1)

Input: `s = "rat"`, `t = "car"`  
Output: `false`  
Объяснение: Разные наборы символов

Input: `s = "listen"`, `t = "silent"`  
Output: `true`  
Объяснение: Одинаковые символы, разные порядки

Input: `s = "hello"`, `t = "hello"`  
Output: `true`  
Объяснение: Идентичные строки

---

## Подход: Подсчёт символов с хеш-таблицей (Character Count + Hash Map)

```go
func ValidAnagram(s string, t string) bool {
	// Оптимизация: если длины разные, точно не анаграммы
	if len(s) != len(t) {
		return false
	}

	charCount := make(map[rune]int)

	// Считаем количество каждого символа в первой строке
	for _, ch := range s {
		charCount[ch]++
	}

	// Проверяем вторую строку
	for _, ch := range t {
		// Если символ не найден или закончились его экземпляры
		if charCount[ch] <= 0 {
			return false
		}
		charCount[ch]--
	}

	return true
}
```

**Идея:**

Используем **хеш-таблицу для подсчёта символов**:

1. **Быстрая проверка:** Если длины разные, не анаграммы
2. **Подсчёт:** Создаём словарь и считаем количество каждого символа в `s`
3. **Проверка:** Для каждого символа в `t`:
   - Если его нет в словаре или счётчик <= 0 → не анаграмма
   - Иначе уменьшаем счётчик
4. **Результат:** Если прошли проверку — анаграмма!

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(k)** |

- **Time O(n)**: Проходим каждую строку один раз (2n операций)
- **Space O(k)**: Хеш-таблица содержит до k уникальных символов (k ≤ размер алфавита, обычно O(1) для английского)

---

## Почему эта техника?

### 1. Быстрая проверка длин

```go
if len(s) != len(t) {
    return false  // Ранний выход
}
```

Если длины разные, анаграммы невозможны!

### 2. Однопроход подсчёт

Вместо:
- Сортировки обеих строк: O(n log n)
- Двойного цикла сравнения: O(n²)

Мы используем один проход: O(n)

### 3. Противовес-проверка

Считаем в `s`, проверяем в `t` — это гарантирует одинаковые количества!

## Альтернативные подходы

### 1. Сортировка (O(n log n))

```go
func ValidAnagramSort(s string, t string) bool {
    if len(s) != len(t) {
        return false
    }
    
    sRunes := []rune(s)
    tRunes := []rune(t)
    
    sort.Slice(sRunes, func(i, j int) bool {
        return sRunes[i] < sRunes[j]
    })
    sort.Slice(tRunes, func(i, j int) bool {
        return tRunes[i] < tRunes[j]
    })
    
    return string(sRunes) == string(tRunes)
}
```

**Сложность:** Time O(n log n), Space O(n)

### 3. Подсчёт с массивом (для ASCII)

```go
func ValidAnagramArray(s string, t string) bool {
    if len(s) != len(t) {
        return false
    }
    
    count := [256]int{}  // ASCII размер
    
    for i := 0; i < len(s); i++ {
        count[s[i]]++
        count[t[i]]--
    }
    
    for i := 0; i < len(count); i++ {
        if count[i] != 0 {
            return false
        }
    }
    return true
}
```

**Сложность:** Time O(n), Space O(256) = O(1)

## Ключевые идеи

1. **Анаграмма = одинаковая частота символов**
   - Не важен порядок, важна "мультимножество"

2. **Ранний выход по длине**
   - Простая оптимизация, часто экономит время

3. **Противовес-подсчёт**
   - Считаем в одной, проверяем в другой
   - Гарантирует корректность

4. **Универсальность**
   - Работает с Unicode символами
   - Работает со специальными символами (пробелы, цифры)

## Интересный факт

Слово **"анаграмма"** происходит из древнегреческого:
- **ana** — "снова, заново"
- **gramma** — "буква, письмо"

То есть буквально "переписанное письмо"!

Известные примеры анаграмм:
- listen ↔ silent
- astronomer ↔ moon starer
- evil ↔ vile
