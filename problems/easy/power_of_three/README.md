# Power of Three

## Условие

Дано целое число `n`. Определить, является ли `n` степенью числа 3.

Целое число `n` является степенью 3, если существует целое число `x` такое, что `n == 3^x`.

## Примеры

Input: `n = 27`  
Output: `true`  
Объяснение: 3^3 = 27

Input: `n = 0`  
Output: `false`

Input: `n = 9`  
Output: `true`  
Объяснение: 3^2 = 9

Input: `n = 10`  
Output: `false`

---

## Два подхода

В этом решении реализованы **два способа**:

### 1. PowerOfThree (Деление в цикле)

Делим число на 3 в цикле, пока оно делится нацело:
- Если в конце получилось 1 → число степень 3
- Если в какой-то момент остаток не равен 0 → не степень 3

**Сложность:** Time O(log₃ n), Space O(1)

### 2. PowerOfThreeByBit (Математический трюк)

Используем знание, что максимальная степень 3 в диапазоне 32-битного целого = 3^19 = 1162261467

Логика:
- Если n является степенью 3, то оно должно делить 3^19 нацело
- Это работает, потому что 3^19 содержит все степени 3 до себя в качестве делителей
- Например: 3^2 = 9 делит 3^19 = 1162261467 нацело (1162261467 / 9 = 129140163)

**Сложность:** Time O(1), Space O(1) — одна операция!

**Примеры:**
- 27 делит 1162261467? 1162261467 / 27 = 43046721 ✓
- 10 делит 1162261467? Остаток ≠ 0 ✗

---

## Сравнение подходов

| Решение | Время | Память | Примечание |
|---|---|---|---|
| Деление в цикле | O(log₃ n) | O(1) | Стандартный подход |
| Математический трюк | O(1) | O(1) | Очень быстро, но специфичен для степеней 3 |

## Идея

Второй подход — это классический трюк для проверки степеней фиксированного числа в ограниченном диапазоне. Вместо цикла мы используем знание о максимально возможном значении (3^19 для int32) и проверяем делимость. Это работает потому, что степени 3 имеют уникальное свойство: если x — степень 3 и x ≤ 3^19, то x делит 3^19 нацело.
