# Happy Number

## Условие

Напишите алгоритм, чтобы определить, является ли число "счастливым" (happy number).

**Счастливое число** определяется следующим процессом:
1. Начиная с положительного целого числа, замените число на сумму квадратов его цифр
2. Повторяйте процесс до тех пор, пока число не станет равно 1 (счастливое число) или оно зацикливается бесконечно в цикле, который не включает 1 (несчастливое число)

## Примеры

Input: `n = 19`  
Output: `true`  
Объяснение:
- 1² + 9² = 1 + 81 = 82
- 8² + 2² = 64 + 4 = 68
- 6² + 8² = 36 + 64 = 100
- 1² + 0² + 0² = 1

Input: `n = 2`  
Output: `false`  
Объяснение: Число зацикливается в цикле 4 → 16 → 37 → 58 → 89 → 145 → 42 → 20 → 4

Input: `n = 7`  
Output: `true`

## Подход

- Используем множество (map) для отслеживания чисел, которые мы уже видели
- Для каждого числа вычисляем сумму квадратов его цифр
- Если получилось 1 → число счастливо
- Если число повторилось (есть в map) → число несчастливо (цикл)
- Иначе добавляем число в map и повторяем процесс

## Сложность

- Time: O(log n) в среднем — количество итераций зависит от числа цифр
- Space: O(1) — размер множества ограничен, так как числа быстро сходятся к малым значениям

## Идея

Ключевая идея — использовать множество для обнаружения циклов. Если мы встретим число, которое уже видели, значит мы в цикле, и число несчастливо. Счастливые числа достаточно редкие: среди первых 100 чисел всего 19 счастливых.
