# Intersection of Two Arrays

## Условие

Найти пересечение двух массивов — элементы, которые присутствуют в **обоих** массивах. Каждый элемент в результате должен быть **уникальным**.

## Примеры

Input: `nums1 = [1, 2, 2, 1]`, `nums2 = [2, 2]`  
Output: `[2]`

Input: `nums1 = [4, 9, 5]`, `nums2 = [9, 4, 9, 8, 4]`  
Output: `[4, 9]` или `[9, 4]`

Input: `nums1 = [1, 2, 3]`, `nums2 = [4, 5, 6]`  
Output: `[]`

---

## Решение: Hash Set

```go
func InsersectionOfTwoArrays(nums1 []int, nums2 []int) []int {
	seen := make(map[int]bool)
	var result []int
	
	// Добавляем все элементы nums1 в множество
	for _, v := range nums1 {
		seen[v] = true
	}
	
	// Проверяем nums2 и собираем пересечение
	for _, v := range nums2 {
		if seen[v] {
			result = append(result, v)
			seen[v] = false  // Исключаем дубликаты
		}
	}
	
	return result
}
```

**Идея:** 
1. Создаём множество из первого массива (O(1) поиск)
2. Для каждого элемента второго массива проверяем наличие в множестве
3. Флаг `seen[v] = false` предотвращает добавление дубликатов

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | O(n + m) |
| **Space** | O(min(n, m)) |

Один проход по каждому массиву, хеш-таблица содержит элементы из меньшего массива.

---

## Альтернативы

### Сортировка обоих массивов (O((n+m) log(n+m)))
```go
sort.Ints(nums1)
sort.Ints(nums2)
// Два указателя для поиска пересечения
```
Медленнее, но использует O(1) памяти (не считая output).

### Два указателя на отсортированных (O(n + m))
Если массивы уже отсортированы, можно использовать два указателя без хеш-таблицы.
