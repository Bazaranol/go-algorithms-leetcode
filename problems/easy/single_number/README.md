# Single Number

## Условие

Дан массив `nums`, в котором каждый элемент появляется **дважды**, кроме одного элемента, который появляется **один раз**.

Найти элемент, который появляется один раз.

**Ограничения:**
- Каждый элемент, кроме одного, появляется ровно 2 раза
- Ровно один элемент появляется один раз
- Требуется линейная временная сложность без использования дополнительной памяти (для второго подхода)

## Примеры

Input: `nums = [2, 2, 1]`  
Output: `1`  
Объяснение: Число 2 появляется дважды, число 1 один раз.

Input: `nums = [4, 1, 2, 1, 2]`  
Output: `4`  
Объяснение: Числа 1 и 2 появляются дважды, число 4 один раз.

Input: `nums = [1]`  
Output: `1`  
Объяснение: Массив содержит только одно число.

Input: `nums = [0, 1, 0, 1, 2]`  
Output: `2`  
Объяснение: Числа 0 и 1 появляются дважды, число 2 один раз.

---

## Два подхода

В этом решении реализованы **два способа**:

### 1. SingleNumber (Хеш-таблица)

Используем хеш-таблицу (map) для подсчета количества появлений каждого элемента:
- Проходим по массиву и считаем количество каждого числа
- Затем находим число с количеством = 1

```go
func SingleNumber(nums []int) int {
    count := make(map[int]int)
    
    for _, num := range nums {
        count[num]++
    }
    
    for num, cnt := range count {
        if cnt == 1 {
            return num
        }
    }
    return 0
}
```

**Сложность:** Time O(n), Space O(n) — используем дополнительную память для хеш-таблицы

### 2. SingleNumberByXOR (Битовая операция XOR)

Используем свойства операции XOR (исключающее ИЛИ):

```go
func SingleNumberByXOR(nums []int) int {
    result := 0
    for _, num := range nums {
        result ^= num
    }
    return result
}
```

**Почему это работает:**

XOR имеет три ключевых свойства:

1. **Самоаннулирование:** `a ^ a = 0`
2. **Нейтральный элемент:** `a ^ 0 = a`
3. **Коммутативность:** `a ^ b = b ^ a`

**Пример для nums = [4, 1, 2, 1, 2]:**

```
Начало: result = 0
result ^= 4  →  0 ^ 4 = 4
result ^= 1  →  4 ^ 1 = 5 (0100 ^ 0001 = 0101)
result ^= 2  →  5 ^ 2 = 7 (0101 ^ 0010 = 0111)
result ^= 1  →  7 ^ 1 = 6 (0111 ^ 0001 = 0110)
result ^= 2  →  6 ^ 2 = 4 (0110 ^ 0010 = 0100)
```

Все дублирующиеся числа аннулируются (дают 0), остается только 4.

**Пошагово с двоичной нотацией:**

| Итерация | Число | Двоичная | result (двоичная) | result (десятичная) |
|----------|-------|----------|-------------------|-------------------|
| 0 | - | - | 0000 | 0 |
| 1 | 4 | 0100 | 0100 | 4 |
| 2 | 1 | 0001 | 0101 | 5 |
| 3 | 2 | 0010 | 0111 | 7 |
| 4 | 1 | 0001 | 0110 | 6 |
| 5 | 2 | 0010 | 0100 | 4 ✓ |

**Сложность:** Time O(n), Space O(1) — идеально оптимально!

---

## Сравнение подходов

| Решение | Время | Память | Примечание |
|---|---|---|---|
| Хеш-таблица | O(n) | O(n) | Простое, понятное, но требует доп. памяти |
| XOR | O(n) | O(1) | **Оптимально**, но требует понимания битовых операций |

## Идея

Ключевая идея XOR подхода — это использование свойства самоаннулирования: если число появляется дважды, XOR этого числа с собой дает 0. Поэтому, XOR-ируя все числа в массиве, все дублирующиеся числа исчезают, и остается только одно число.

Это один из самых элегантных примеров использования битовых операций в алгоритмике!
