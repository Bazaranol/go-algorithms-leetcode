# Missing Number

## Условие

Дан массив `nums` длины `n`, содержащий числа от 0 до n **включительно**. Ровно одно число отсутствует.

Найти пропущенное число.

**Ограничения:**
- Массив содержит числа от 0 до n в произвольном порядке
- Ровно одно число пропущено (может быть 0 или n)

## Примеры

Input: `nums = [3, 0, 1]`  
Output: `2`  
Объяснение: n = 3, числа от 0 до 3 это {0, 1, 2, 3}. Число 2 пропущено.

Input: `nums = [0, 1]`  
Output: `2`  
Объяснение: n = 2, числа от 0 до 2 это {0, 1, 2}. Число 2 пропущено.

Input: `nums = [9, 6, 4, 2, 3, 5, 7, 0, 1]`  
Output: `8`  
Объяснение: n = 9, число 8 пропущено.

Input: `nums = [0]`  
Output: `1`  
Объяснение: n = 1, число 1 пропущено.

## Подход

Используется математическая формула для суммы арифметической последовательности:

**Сумма чисел от 0 до n = n × (n + 1) / 2**

Алгоритм:
1. Вычислить ожидаемую сумму всех чисел от 0 до n: `expected = n * (n + 1) / 2`
2. Вычислить актуальную сумму чисел в массиве: `actual = сумма всех элементов`
3. Пропущенное число = `expected - actual`

**Пример для nums = [3, 0, 1]:**
- n = 3
- expected = 3 × 4 / 2 = 6 (сумма 0 + 1 + 2 + 3)
- actual = 3 + 0 + 1 = 4
- missing = 6 - 4 = 2 ✓

## Сложность

- Time: O(n) — один проход по массиву для вычисления суммы
- Space: O(1) — используем только несколько переменных, без дополнительных структур данных

## Идея

Ключевая идея — использовать математическую формулу вместо хеш-таблицы или сортировки. Это даёт нам оптимальную временную и пространственную сложность.

**Почему это работает:**

Сумма 0 + 1 + 2 + ... + n — это хорошо известная формула Гаусса:
```
S = n × (n + 1) / 2
```

Если из полного набора удалить одно число x, то новая сумма будет S - x. Поэтому:
```
missing = (expected sum) - (actual sum)
```

## Альтернативные подходы

**1. Хеш-таблица (Hash Set)**
```go
func MissingNumber(nums []int) int {
    seen := make(map[int]bool)
    for _, v := range nums {
        seen[v] = true
    }
    for i := 0; i <= len(nums); i++ {
        if !seen[i] {
            return i
        }
    }
    return -1
}
```
- Time: O(n)
- Space: O(n)

**2. Сортировка**
```go
func MissingNumber(nums []int) int {
    sort.Ints(nums)
    for i := 0; i <= len(nums); i++ {
        if i != nums[i] {
            return i
        }
    }
    return -1
}
```
- Time: O(n log n)
- Space: O(1)

**3. XOR (битовый трюк)**
```go
func MissingNumber(nums []int) int {
    result := len(nums)
    for i, v := range nums {
        result ^= i ^ v
    }
    return result
}
```
- Time: O(n)
- Space: O(1)
- XOR имеет свойство: a ^ a = 0 и a ^ 0 = a

## Сравнение подходов

| Решение | Time | Space | Примечание |
|---|---|---|---|
| Математическая формула | O(n) | O(1) | **Оптимально** |
| Хеш-таблица | O(n) | O(n) | Простая, но требует доп. памяти |
| Сортировка | O(n log n) | O(1) | Медленнее, изменяет массив |
| XOR | O(n) | O(1) | Умный трюк, но сложнее понять |
