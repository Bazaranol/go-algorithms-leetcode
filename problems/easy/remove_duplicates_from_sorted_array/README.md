# Remove Duplicates from Sorted Array

## Условие

Дан **отсортированный** массив целых чисел `nums`. Нужно удалить дубликаты **in-place** таким образом, чтобы каждый элемент появлялся **ровно один раз**.

Функция должна вернуть количество уникальных элементов, и первые `k` элементов массива должны содержать уникальные значения в исходном порядке.

**Ограничения:**
- Массив уже отсортирован
- Нужно модифицировать массив in-place (без использования дополнительного массива)
- Вернуть количество уникальных элементов

## Примеры

Input: `nums = [1, 1, 2]`  
Output: `2, nums = [1, 2, _]`  
Объяснение: Функция возвращает 2, первые 2 элемента массива — [1, 2]

Input: `nums = [0, 0, 1, 1, 1, 2, 2, 3, 3, 4]`  
Output: `5, nums = [0, 1, 2, 3, 4, _, _, _, _, _]`  
Объяснение: Функция возвращает 5, первые 5 элементов — уникальные значения [0, 1, 2, 3, 4]

Input: `nums = [1, 2, 3, 4, 5]`  
Output: `5, nums = [1, 2, 3, 4, 5]`  
Объяснение: Нет дубликатов, все 5 элементов уникальны

---

## Подход: Двухуказательная техника (Two-Pointer)

```go
func RemoveDuplicatesFromSortedArray(nums []int) int {
	slow := 0

	for fast := 1; fast < len(nums); fast++ {
		if nums[slow] != nums[fast] {
			slow++
			nums[slow] = nums[fast]
		}
	}

	return slow + 1
}
```

**Идея:**

Используем **два указателя** — `slow` и `fast`:

1. **slow** — указатель на позицию, куда нужно поместить следующий уникальный элемент
2. **fast** — указатель для итерации по массиву

**Алгоритм:**

1. Инициализируем `slow = 0` (первый элемент всегда уникален)
2. Проходим по массиву с `fast` от 1 до конца
3. Если `nums[slow] != nums[fast]`:
   - Это новый уникальный элемент
   - Увеличиваем `slow`
   - Копируем значение `nums[fast]` в `nums[slow]`
4. Возвращаем `slow + 1` (количество уникальных элементов)

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

- **Time O(n)**: Один проход по массиву, каждый элемент посещается один раз
- **Space O(1)**: Используем только два указателя, без дополнительных структур данных

---

## Ключевые идеи

1. **Отсортированность — ключевой фактор**: Благодаря тому, что массив отсортирован, дубликаты находятся рядом друг с другом. Это позволяет обнаружить их с помощью простого сравнения соседних элементов.

2. **In-place модификация**: Мы не создаем новый массив, а переписываем элементы в существующий. Это экономит память.

3. **Два указателя**: `slow` отслеживает позицию для следующего уникального элемента, `fast` ищет новые уникальные значения.

4. **Не нужно очищать хвост**: Нам не нужно заботиться о значениях после позиции `slow`. Важно только то, что первые `k` элементов содержат уникальные значения.

## Альтернативный подход (с использованием памяти)

Если бы требование в-place не было, можно было бы использовать set или map:

```go
func RemoveDuplicatesAlternative(nums []int) int {
    seen := make(map[int]bool)
    k := 0
    
    for _, num := range nums {
        if !seen[num] {
            seen[num] = true
            nums[k] = num
            k++
        }
    }
    
    return k
}
```

**Сложность**: Time O(n), Space O(n)

Но двухуказательный подход **лучше** благодаря O(1) дополнительной памяти!
