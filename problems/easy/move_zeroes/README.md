# Move Zeroes

## Условие

Дан массив целых чисел `nums`. Нужно переместить все нули в **конец массива**, сохраняя **относительный порядок** ненулевых элементов.

**Важно:**
- Нужно модифицировать массив in-place (без создания нового массива)
- Относительный порядок ненулевых элементов должен быть сохранен

## Примеры

Input: `nums = [0, 1, 0, 3, 12]`  
Output: `[1, 3, 12, 0, 0]`  
Объяснение: Все нули переместились в конец, порядок [1, 3, 12] сохранен

Input: `nums = [0]`  
Output: `[0]`  
Объяснение: Массив состоит из одного нуля

Input: `nums = [1, 2, 3]`  
Output: `[1, 2, 3]`  
Объяснение: Нет нулей, массив не изменился

Input: `nums = [0, 0, 1]`  
Output: `[1, 0, 0]`  
Объяснение: Нули переместились в конец

---

## Подход: Двухуказательная техника (Two-Pointer)

```go
func MoveZeroes(nums []int) []int {
	slow := 0
	fast := 0

	for fast < len(nums) {
		if nums[fast] != 0 {
			nums[slow], nums[fast] = nums[fast], nums[slow]
			slow++
		}
		fast++
	}
	return nums
}
```

**Идея:**

Используем **два указателя** — `slow` и `fast`:

1. **slow** — указатель на позицию, где должен находиться следующий ненулевой элемент
2. **fast** — указатель для сканирования всего массива

**Алгоритм:**

1. Инициализируем `slow = 0`, `fast = 0`
2. Проходим по массиву с `fast` от 0 до конца
3. Если `nums[fast] != 0` (нашли ненулевой элемент):
   - Меняем местами `nums[slow]` и `nums[fast]`
   - Увеличиваем `slow`
4. Всегда увеличиваем `fast`
5. В результате все ненулевые элементы окажутся в начале в правильном порядке, нули — в конце

---

## Пошаговый пример

**Входные данные:** `nums = [0, 1, 0, 3, 12]`

| Шаг | fast | nums[fast] | Условие | Действие | slow | Состояние массива |
|-----|------|-----------|---------|----------|------|-------------------|
| 0 | - | - | - | Инициализация | 0 | [0, 1, 0, 3, 12] |
| 1 | 0 | 0 | == 0 | Пропускаем | 0 | [0, 1, 0, 3, 12] |
| 2 | 1 | 1 | != 0 | swap(0,1), slow++ | 1 | [**1**, 0, 0, 3, 12] |
| 3 | 2 | 0 | == 0 | Пропускаем | 1 | [1, 0, 0, 3, 12] |
| 4 | 3 | 3 | != 0 | swap(1,3), slow++ | 2 | [1, **3**, 0, 0, 12] |
| 5 | 4 | 12 | != 0 | swap(2,4), slow++ | 3 | [1, 3, **12**, 0, 0] |

**Результат:** `[1, 3, 12, 0, 0]` ✓

## Визуализация

```
Начальное состояние (slow = 0, fast = 0):
nums = [0, 1, 0, 3, 12]
        ↑
    slow/fast

Итерация 1 (fast = 1): nums[1] = 1 != 0 → swap(0,1)
nums = [1, 0, 0, 3, 12]
        ↑
     slow

Итерация 2 (fast = 2): nums[2] = 0 == 0 → пропускаем
nums = [1, 0, 0, 3, 12]
        ↑
     slow

Итерация 3 (fast = 3): nums[3] = 3 != 0 → swap(1,3)
nums = [1, 3, 0, 0, 12]
           ↑
        slow

Итерация 4 (fast = 4): nums[4] = 12 != 0 → swap(2,4)
nums = [1, 3, 12, 0, 0]
              ↑
           slow

Финальный результат: [1, 3, 12, 0, 0] ✓
Все нули в конце, порядок [1, 3, 12] сохранен
```

## Ключевые моменты

1. **Разделение на две части**: 
   - До позиции `slow` — ненулевые элементы
   - После позиции `slow` — обработанные элементы (могут быть нули)

2. **Swap вместо удаления**: 
   - Используем обмен, а не удаление элемента
   - Это позволяет избежать O(n²) сложности

3. **In-place модификация**: 
   - Не создаем новый массив
   - Работаем с исходным массивом

4. **Сохранение порядка**: 
   - Мы не меняем порядок ненулевых элементов
   - Обрабатываем их по очереди слева направо

---

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

- **Time O(n)**: Один проход по массиву, каждый элемент посещается один раз
- **Space O(1)**: Используем только два указателя (slow и fast), без дополнительных структур данных

---

## Почему именно эта техника?

**Наивный подход** (удаление нулей):
```go
func MoveZeroesNaive(nums []int) []int {
    for i := 0; i < len(nums); i++ {
        if nums[i] == 0 {
            nums = append(nums[:i], nums[i+1:]...)  // O(n) удаление
            i--
        }
    }
    return nums
}
```
**Сложность**: O(n²) — очень неэффективно!

**Наш подход** (swap):
- Используем только swaps, каждая операция O(1)
- Всего O(n) операций
- **Идеально оптимально!**

---

## Сравнение с удалением

| Операция | Сложность | Пояснение |
|----------|-----------|-----------|
| Удаление элемента | O(n) | Нужно сдвинуть все элементы справа |
| Swap элементов | O(1) | Просто обмениваем два элемента |
| Наш алгоритм (n swaps) | O(n) | Все swaps вместе |
| Наивный (n удалений) | O(n²) | Каждое удаление стоит O(n) |

---

## Интересный факт

Алгоритм работает так же эффективно, является ли массив:
- Полностью заполнен нулями → один левый указатель медленно движется
- Без нулей → просто копирует элемент на себя (swap с самим собой)
- Смешанный → оптимальный баланс

Это показывает **элегантность дизайна алгоритма**!
