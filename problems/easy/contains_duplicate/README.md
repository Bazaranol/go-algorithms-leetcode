# Contains Duplicate

## Условие

Дан массив целых чисел `nums`. Определить, содержит ли массив **любой дубликат**.

Возвращает:
- `true` — если массив содержит хотя бы один дубликат
- `false` — если все элементы уникальны

**Требования:**
- Найти наличие дубликата (не сами дубликаты)
- Работать с отрицательными числами
- Работать с нулём
- Работать с большими числами

## Примеры

Input: `nums = [1, 2, 3, 1]`  
Output: `true`  
Объяснение: Число 1 повторяется на индексах 0 и 3

Input: `nums = [1, 2, 3, 4]`  
Output: `false`  
Объяснение: Все элементы уникальны

Input: `nums = [1, 2, 2, 3]`  
Output: `true`  
Объяснение: Число 2 повторяется

Input: `nums = [1]`  
Output: `false`  
Объяснение: Один элемент не может иметь дубликат

---

## Подход: Хеш-таблица (Hash Set)

```go
func ContainsDuplicate(nums []int) bool {
	seen := make(map[int]bool)

	for _, v := range nums {
		if seen[v] {
			return true
		}
		seen[v] = true
	}
	return false
}
```

**Идея:**

Используем **хеш-таблицу как множество (set)** для отслеживания видимых элементов:

1. **Инициализация:** Создаём пустую карту
2. **Итерация:** Для каждого элемента в массиве:
   - Если элемент уже в карте → нашли дубликат, возвращаем `true`
   - Иначе добавляем элемент в карту
3. **Результат:** Если прошли весь массив → дубликатов нет, возвращаем `false`

**Ключевая идея:**

Хеш-таблица позволяет проверить наличие элемента за **O(1)** время вместо линейного поиска O(n).

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(n)** |

- **Time O(n)**: Один проход по массиву, каждый элемент проверяется один раз
- **Space O(n)**: Хеш-таблица может содержать до n элементов в худшем случае (когда все элементы уникальны)

---

## Почему эта техника?

### 1. Оптимальность

```go
// ✓ O(n) — ЛУЧШЕ ВСЕГО!
seen := make(map[int]bool)
```

Не можем做得лучше чем O(n), так как нужно посмотреть каждый элемент минимум один раз.

### 2. Однопроход проверка

В отличие от:
- **Сортировки:** O(n log n) — медленнее
- **Двойного цикла:** O(n²) — намного медленнее

Мы проверяем в один проход: **O(n)**

### 3. Ранний выход

Возвращаем `true` **сразу** при обнаружении первого дубликата:

```go
if seen[v] {
    return true  // Ранний выход!
}
```

Это даёт нам на практике ускорение в случайных данных.

## Альтернативные подходы

### 1. Сортировка (O(n log n))

```go
func ContainsDuplicateSort(nums []int) bool {
    // Сортируем и ищем соседние дубликаты
    sort.Ints(nums)
    
    for i := 0; i < len(nums)-1; i++ {
        if nums[i] == nums[i+1] {
            return true
        }
    }
    return false
}
```

**Сложность:** Time O(n log n), Space O(1) или O(n) (зависит от сортировки)

**Минусы:**
- Медленнее из-за сортировки
- Изменяет исходный массив (если не сделать копию)

### 2. Брут-форс (O(n²)) ✗

```go
func ContainsDuplicateBruteForce(nums []int) bool {
    // Для каждого элемента проверяем все остальные
    for i := 0; i < len(nums); i++ {
        for j := i+1; j < len(nums); j++ {
            if nums[i] == nums[j] {
                return true
            }
        }
    }
    return false
}
```

**Сложность:** Time O(n²), Space O(1)

**Минусы:**
- Очень медленно даже для малых массивов
- Вложенные циклы = квадратичная сложность

- ✓ Работает со всеми числами (отрицательные, ноль, большие)

## Ключевые идеи

1. **Множество (Set) паттерн**
   - Используем хеш-таблицу как множество
   - Быстрая проверка наличия: O(1)
   - Нет нужды считать частоты (как в Valid Anagram)

2. **Ранний выход**
   - Возвращаем true сразу при первом дубликате
   - На практике часто быстрее среднего случая

3. **Универсальность**
   - Работает с любыми целыми числами
   - Отрицательные числа не проблема
   - Ноль обрабатывается корректно

4. **Один проход**
   - Одна итерация по массиву
   - Никаких вложенных циклов

---

## Интересный факт

**Принцип Дирихле (Pigeonhole Principle):**

Если в массиве из n элементов все элементы уникальны, то максимум мы можем хранить **n уникальных значений**.

Как только мы видим элемент, который уже видели, мы получили дубликат — это гарантировано!
