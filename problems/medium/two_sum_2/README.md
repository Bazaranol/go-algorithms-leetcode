# Two Sum II - Input Array Is Sorted

## Условие

Дан **отсортированный массив целых чисел** `numbers` и целевое число `target`. Нужно найти **два различных индекса** `i` и `j` таких, что `numbers[i] + numbers[j] == target`.

**Особенности:**
- Массив уже отсортирован (в порядке возрастания)
- Нужно вернуть 1-индексированные значения (не 0-индексированные!)
- Гарантируется, что решение существует (ровно одна пара)
- Нельзя использовать один и тот же элемент дважды

## Примеры

Input: `numbers = [2, 7, 11, 15]`, `target = 9`  
Output: `[1, 2]`  
Объяснение: numbers[0] + numbers[1] = 2 + 7 = 9, вернуть [1, 2] (1-индексированные)

Input: `numbers = [2, 3, 4]`, `target = 6`  
Output: `[1, 3]`  
Объяснение: 2 + 4 = 6, индексы 1 и 3 (1-индексированные)

Input: `numbers = [-1, 0]`, `target = -1`  
Output: `[1, 2]`  
Объяснение: -1 + 0 = -1

---

## Подход: Двухуказательная техника (Two-Pointer)

```go
func TwoSum2(numbers []int, target int) []int {
	left := 0
	right := len(numbers) - 1

	for numbers[left]+numbers[right] != target {
		if numbers[left]+numbers[right] < target {
			left++
		}
		if numbers[left]+numbers[right] > target {
			right--
		}
	}
	return []int{left + 1, right + 1}
}
```

**Идея:**

Используем два указателя, начиная с противоположных концов **отсортированного массива**:

1. **left** — указатель на начало массива (минимальное значение)
2. **right** — указатель на конец массива (максимальное значение)

**Алгоритм:**

1. Вычисляем сумму `numbers[left] + numbers[right]`
2. Если сумма равна target → вернуть индексы (с +1 для 1-индексирования)
3. Если сумма меньше target → нужно большее число, двигаем `left` вправо
4. Если сумма больше target → нужно меньшее число, двигаем `right` влево
5. Повторяем, пока не найдем решение

**Ключевая идея:** 
- Потому что массив отсортирован, мы можем **выбросить** половину вариантов на каждом шаге
- Если сумма слишком маленькая, все пары с текущим `right` тоже маленькие
- Если сумма слишком большая, все пары с текущим `left` тоже большие

---

## Пошаговый пример

**Входные данные:** `numbers = [2, 7, 11, 15]`, `target = 9`

| Шаг | left | right | numbers[left] | numbers[right] | Сумма | Сравнение | Действие |
|-----|------|-------|---------------|----------------|-------|-----------|----------|
| 0 | 0 | 3 | 2 | 15 | 17 | 17 > 9 | right-- |
| 1 | 0 | 2 | 2 | 11 | 13 | 13 > 9 | right-- |
| 2 | 0 | 1 | 2 | 7 | 9 | 9 == 9 | ✓ Найдено! |

**Результат:** `[1, 2]` (возвращаем [left+1, right+1] = [0+1, 1+1])

---

## Визуализация

```
Начальное состояние:
numbers = [2, 7, 11, 15]
          ↑           ↑
        left        right
        Сумма = 2 + 15 = 17 > 9 → right--

Шаг 2:
numbers = [2, 7, 11, 15]
          ↑        ↑
        left     right
        Сумма = 2 + 11 = 13 > 9 → right--

Шаг 3:
numbers = [2, 7, 11, 15]
          ↑    ↑
        left right
        Сумма = 2 + 7 = 9 == 9 ✓
        
Результат: [1, 2]
```

## Пошагово с более сложным примером

**Входные данные:** `numbers = [1, 2, 3, 4, 5, 6, 7]`, `target = 11`

| Шаг | left | right | Сумма | Результат | Причина |
|-----|------|-------|-------|-----------|---------|
| 0 | 0 | 6 | 1+7=8 | 8 < 11 | Нужно больше, left++ |
| 1 | 1 | 6 | 2+7=9 | 9 < 11 | Нужно больше, left++ |
| 2 | 2 | 6 | 3+7=10 | 10 < 11 | Нужно больше, left++ |
| 3 | 3 | 6 | 4+7=11 | 11 == 11 | ✓ Найдено! |

**Результат:** `[4, 7]` (возвращаем [3+1, 6+1])

## Сравнение с Two Sum (неотсортированный массив)

**Two Sum (LeetCode 1)** — массив не отсортирован:
- Используем hash map для хранения элементов
- Time: O(n), Space: O(n)

```go
func TwoSum(nums []int, target int) []int {
    seen := make(map[int]int)
    for i, num := range nums {
        complement := target - num
        if idx, ok := seen[complement]; ok {
            return []int{idx, i}
        }
        seen[num] = i
    }
    return nil
}
```

**Two Sum II** — массив отсортирован:
- Используем two-pointer технику
- Time: O(n), Space: **O(1)** — лучше!

```go
func TwoSum2(numbers []int, target int) []int {
    left, right := 0, len(numbers)-1
    for numbers[left]+numbers[right] != target {
        if numbers[left]+numbers[right] < target {
            left++
        } else {
            right--
        }
    }
    return []int{left+1, right+1}
}
```

---

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

- **Time O(n)**: В худшем случае проходим весь массив один раз (один указатель достигает конца)
- **Space O(1)**: Используем только два указателя, без дополнительных структур данных

---

## Почему это работает?

**Ключевое свойство отсортированного массива:**

Если sum = `numbers[left] + numbers[right]`:

1. **sum < target:**
   - Все пары `(left, right), (left, right-1), ... (left, 0)` тоже < target
   - Нет смысла уменьшать `right`
   - Нужно увеличить `sum` → двигаем `left` вправо

2. **sum > target:**
   - Все пары `(left, right), (left+1, right), ... (n-1, right)` тоже > target
   - Нет смысла увеличивать `left`
   - Нужно уменьшить `sum` → двигаем `right` влево

3. **sum == target:**
   - Нашли ответ!

Это элегантный пример использования отсортированности для оптимизации!

---

## Интересный момент: Индексирование

**Почему возвращаем [left+1, right+1]?**

LeetCode требует **1-индексированный результат**, а массивы в Go 0-индексированы:

```
0-индексированные:  [2, 7, 11, 15]
                      0  1  2   3

1-индексированные:  [2, 7, 11, 15]
                      1  2  3   4
```

Поэтому для элементов на позициях 0 и 1 (0-индекс) вернуть [1, 2] (1-индекс).
