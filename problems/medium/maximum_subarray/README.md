# Maximum Subarray

## Условие

Дан массив целых чисел `nums`. Нужно найти **смежный подмассив** (контигуозный массив) с наибольшей суммой и вернуть эту сумму.

**Смежный подмассив** — это непрерывная последовательность элементов из исходного массива.

**Ограничения:**
- Массив не пуст
- Минимум один элемент в массиве

## Примеры

Input: `nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]`  
Output: `6`  
Объяснение: Подмассив [4, -1, 2, 1] имеет наибольшую сумму = 6

Input: `nums = [5, 4, -1, 7, 8]`  
Output: `23`  
Объяснение: Весь массив = 5 + 4 + (-1) + 7 + 8 = 23

Input: `nums = [-1]`  
Output: `-1`  
Объяснение: Только один элемент, это наибольшая возможная сумма

Input: `nums = [-5, -2, -8, -1, -4]`  
Output: `-1`  
Объяснение: Наибольший элемент = -1

---

## Подход: Алгоритм Кадане (Kadane's Algorithm)

```go
func MaximumSubarray(nums []int) int {
	localMax := nums[0]
	globalMax := nums[0]

	for i := 1; i < len(nums); i++ {
		localMax = max(localMax+nums[i], nums[i])
		if localMax > globalMax {
			globalMax = max(localMax, globalMax)
		}
	}

	return globalMax
}
```

**Идея:**

Алгоритм Кадане использует **динамическое программирование**:

1. **localMax** — максимальная сумма подмассива, **заканчивающегося** на текущей позиции
2. **globalMax** — максимальная сумма найденная **до сих пор**

На каждом шаге принимаем решение:
- Продолжить текущий подмассив: `localMax + nums[i]`
- Или начать новый подмассив с текущего элемента: `nums[i]`

Выбираем вариант с большей суммой.

---

## Пошаговый пример

**Входные данные:** `nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]`

| i | nums[i] | localMax = max(localMax+nums[i], nums[i]) | globalMax | Пояснение |
|---|---------|----------------------------------------|-----------|-----------|
| - | - | -2 | -2 | Инициализация |
| 1 | 1 | max(-2+1=-1, 1)=**1** | max(-2, 1)=**1** | Начали новый подмассив |
| 2 | -3 | max(1-3=-2, -3)=**-2** | 1 | Продолжаем (хоть и отрицательно) |
| 3 | 4 | max(-2+4=2, 4)=**4** | max(1, 4)=**4** | Начали новый с 4 |
| 4 | -1 | max(4-1=3, -1)=**3** | 4 | Продолжаем подмассив [4, -1] |
| 5 | 2 | max(3+2=5, 2)=**5** | max(4, 5)=**5** | Продолжаем [4, -1, 2] |
| 6 | 1 | max(5+1=6, 1)=**6** | max(5, 6)=**6** | Продолжаем [4, -1, 2, 1] ✓ |
| 7 | -5 | max(6-5=1, -5)=**1** | 6 | Продолжаем (хоть ухудшилось) |
| 8 | 4 | max(1+4=5, 4)=**5** | 6 | Начали новый [4] |

**Результат:** globalMax = 6, подмассив [4, -1, 2, 1]

## Ключевые моменты

1. **Локальный максимум**: На каждой позиции мы решаем, нужно ли добавить текущий элемент к предыдущему подмассиву или начать заново.

2. **Прерывание подмассива**: Если добавление элемента к подмассиву делает его меньше самого элемента, лучше начать новый подмассив.

3. **Глобальный максимум**: Отслеживаем лучший результат, найденный когда-либо.

---

## Почему это работает?

Алгоритм основан на наблюдении, что оптимальный подмассив либо:
- Расширяет оптимальный подмассив до позиции i-1
- Или начинается с позиции i

Это свойство оптимальной подструктуры (optimal substructure) характерно для задач динамического программирования.

---

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

- **Time O(n)**: Один проход по массиву, каждый элемент обрабатывается один раз
- **Space O(1)**: Используем только две переменные (localMax и globalMax), без дополнительных структур

---

## Альтернативный подход (Divide and Conquer)

Можно решить задачу методом "разделяй и властвуй":
- Разделить массив пополам
- Найти максимум в левой половине
- Найти максимум в правой половине
- Найти максимум, пересекающий середину
- Вернуть максимум из трех

**Сложность**: Time O(n log n), Space O(log n) для рекурсии

Алгоритм Кадане **намного лучше** — O(n)!

## Исторический факт

Алгоритм был разработан **Джеем Кадане** (Jay Kadane) в 1984 году и является одним из классических примеров динамического программирования. Это одна из самых элегантных задач в компьютерных науках.
