# Subarray Sum Equals K

## Условие

Дан массив целых чисел `nums` и целое число `k`. Нужно найти количество **смежных подмассивов**, которые в сумме дают `k`.

**Требования:**
- Найти **количество** (не сами подмассивы)
- Подмассивы должны быть смежными (contiguous)
- Массив может содержать отрицательные числа
- Элементы могут повторяться

## Примеры

Input: `nums = [1, 1, 1], k = 2`  
Output: `2`  
Объяснение: Подмассивы [1,1] (индексы 0-1) и [1,1] (индексы 1-2) оба дают сумму 2

Input: `nums = [1, 2, 3], k = 3`  
Output: `2`  
Объяснение: Подмассивы [3] и [1,2] дают сумму 3

Input: `nums = [0, 0, 0, 0], k = 0`  
Output: `10`  
Объяснение: Все 10 возможных подмассивов (включая весь массив) дают сумму 0

Input: `nums = [1, 2, -2, 3], k = 3`  
Output: `3`  
Объяснение: [1,2], [2,-2,3], [3]

---

## Подход: Префиксная сумма + Хеш-таблица (Prefix Sum + Hash Map)

```go
func SubarraySumEqualsK(nums []int, k int) int {
	curSum, count := 0, 0
	sumMap := make(map[int]int)
	sumMap[0] = 1

	for _, num := range nums {
		curSum += num

		if val, exists := sumMap[curSum-k]; exists {
			count += val
		}

		sumMap[curSum]++
	}
	return count
}
```

**Идея:**

Используем **техника префиксной суммы** с **хеш-таблицей**:

1. **curSum** — накопительная сумма (префиксная сумма)
2. **sumMap** — словарь, хранящий количество каждой префиксной суммы
3. Ключевая идея: Если `curSum - k` существует в карте, значит есть подмассив заканчивающийся в текущей позиции с суммой k!

**Почему это работает:**

Если у нас есть префиксные суммы:
- `prefix[i]` — сумма элементов от начала до индекса i
- `prefix[j]` — сумма элементов от начала до индекса j

Сумма подмассива с индексами (j, i] равна: `prefix[i] - prefix[j]`

Нам нужно: `prefix[i] - prefix[j] = k`  
Переставляем: `prefix[j] = prefix[i] - k`

Поэтому мы ищем в карте значение `curSum - k`!

**Алгоритм:**

1. Инициализируем карту с `{0: 1}` (пустой префикс)
2. Для каждого элемента:
   - Добавляем его к `curSum`
   - Проверяем, есть ли `curSum - k` в карте
   - Если есть, добавляем его количество к счётчику
   - Увеличиваем количество `curSum` в карте

## Пошаговый пример

**Входные данные:** `nums = [1, 1, 1], k = 2`

| i | num | curSum | Ищем | Найдено | count | sumMap | Объяснение |
|---|-----|--------|------|---------|-------|--------|-----------|
| - | - | 0 | - | - | 0 | {0:1} | Инициализация |
| 0 | 1 | 1 | 1-2=-1 | Нет | 0 | {0:1, 1:1} | Нет подмассива |
| 1 | 1 | 2 | 2-2=0 | Да, count=1 | 1 | {0:1, 1:1, 2:1} | Нашли [1,1]! |
| 2 | 1 | 3 | 3-2=1 | Да, count=1 | 2 | {0:1, 1:1, 2:1, 3:1} | Нашли [1,1]! |

**Результат:** count = 2 ✓

Найденные подмассивы:
- На шаге i=1: `curSum=2`, нашли `curSum-k=0`, это означает подмассив от (после позиции с sum=0) до текущей позиции = [1,1]
- На шаге i=2: `curSum=3`, нашли `curSum-k=1`, это означает подмассив от (после позиции с sum=1) до текущей позиции = [1,1]

## Визуализация: Как работают префиксные суммы

```
Массив:      [1, 1, 1]
Индекс:       0  1  2

Префиксные суммы:
    0  1  2  3
┌──┬──┬──┬──┐
│0 │1 │2 │3 │
└──┴──┴──┴──┘
   ↑  ↑  ↑  ↑
  До До До До
  индекса 0, 1, 2, конец

Подмассив (0,1]: сумма = prefix[2] - prefix[0] = 2 - 0 = 2 ✓
Подмассив (1,2]: сумма = prefix[3] - prefix[1] = 3 - 1 = 2 ✓

Ищем подмассивы с суммой 2:
- Когда curSum = 2: ищем 2 - 2 = 0 в карте → найдено! (один раз)
- Когда curSum = 3: ищем 3 - 2 = 1 в карте → найдено! (один раз)
Итого: 2 подмассива ✓
```

## Пример с отрицательными числами

**Входные данные:** `nums = [1, 2, -2, 3], k = 3`

| i | num | curSum | Ищем | Найдено | count | sumMap | Подмассив |
|---|-----|--------|------|---------|-------|--------|-----------|
| - | - | 0 | - | - | 0 | {0:1} | - |
| 0 | 1 | 1 | -2 | Нет | 0 | {0:1, 1:1} | - |
| 1 | 2 | 3 | 0 | Да! | 1 | {0:1, 1:1, 3:1} | [1,2] |
| 2 | -2 | 1 | -2 | Нет | 1 | {0:1, 1:2, 3:1} | - |
| 3 | 3 | 4 | 1 | Да! (count=2) | 3 | {0:1, 1:2, 3:1, 4:1} | [2,-2,3], [3] |

**Результат:** count = 3 ✓

Найденные подмассивы:
1. [1, 2] (индексы 0-1)
2. [2, -2, 3] (индексы 1-3)
3. [3] (индекс 3)

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(n)** |

- **Time O(n)**: Один проход по массиву, каждый элемент обрабатывается один раз
- **Space O(n)**: Хеш-таблица может содержать до n+1 различных префиксных сумм

---

## Почему это лучше чем Brute Force?

### Brute Force подход (O(n²))

```go
func SubarraySumBruteForce(nums []int, k int) int {
    count := 0
    for i := 0; i < len(nums); i++ {
        sum := 0
        for j := i; j < len(nums); j++ {
            sum += nums[j]
            if sum == k {
                count++
            }
        }
    }
    return count
}
```

**Сложность:** O(n²) — проверяем все пары (i, j)

### Наш подход (O(n))

- Только один проход по массиву
- Поиск в хеш-таблице O(1)
- **Всего O(n)**

---

## Ключевые идеи

1. **Префиксная сумма:** Сумма подмассива = префиксная сумма в конце - префиксная сумма в начале

2. **Обратная связь:** Вместо поиска конца, мы ищем начало
   - Нам нужно: `sum[end] - sum[start] = k`
   - Ищем: `sum[start] = sum[end] - k`

3. **Инициализация карты:** `sumMap[0] = 1` нужна для подмассивов, начинающихся с индекса 0

4. **Подсчёт дубликатов:** Если одна и та же префиксная сумма встречается несколько раз, все они могут дать подмассив с нужной суммой

## Сравнение подходов

| Подход | Time | Space | Примечание |
|--------|------|-------|-----------|
| Brute Force | O(n²) | O(1) | ✗ Слишком медленно |
| Prefix Sum + Hash Map | O(n) | O(n) | ✓ **Оптимально!** |
| Кумулятивный массив | O(n²) | O(n) | Промежуточный вариант |

## Интересный факт

Эта техника (префиксная сумма + хеш-таблица) работает для решения многих похожих задач:
- Найти подмассив с суммой = k
- Найти максимальный подмассив с суммой ≤ k (с небольшой модификацией)
- Найти подмассив с суммой, делящейся на m

Это очень универсальная техника в динамическом программировании!
