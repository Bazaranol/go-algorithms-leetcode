# 3Sum

## Условие

Дан массив целых чисел `nums`. Нужно найти **все уникальные тройки** элементов, которые в сумме дают **ноль**.

**Требования:**
- Найти все уникальные тройки (без дубликатов)
- Никакую тройку не повторять
- Не использовать один элемент дважды (если только он не появляется несколько раз в массиве)
- Порядок результата не важен

## Примеры

Input: `nums = [-1, 0, 1, 2, -1, -4]`  
Output: `[[-1, -1, 2], [-1, 0, 1]]`  
Объяснение: 
- -1 + (-1) + 2 = 0
- -1 + 0 + 1 = 0

Input: `nums = [0, 1, 1]`  
Output: `[]`  
Объяснение: Нет тройки с суммой 0

Input: `nums = [-2, 0, 1, 1, 2]`  
Output: `[[-2, 0, 2], [-2, 1, 1]]`  
Объяснение:
- -2 + 0 + 2 = 0
- -2 + 1 + 1 = 0

Input: `nums = [0, 0, 0, 0]`  
Output: `[[0, 0, 0]]`  
Объяснение: Одна уникальная тройка (дубликаты пропускаем)

---

## Подход: Сортировка + Двухуказательная техника

```go
func ThreeSum(nums []int) [][]int {
	sort.Ints(nums)
	var result [][]int

	for i := 0; i < len(nums)-2; i++ {
		// Оптимизация: если первый элемент > 0, сумма не может быть 0
		if nums[i] > 0 {
			break
		}

		// Пропускаем дубликаты
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}

		left := i + 1
		right := len(nums) - 1

		for left < right {
			sum := nums[i] + nums[left] + nums[right]

			if sum == 0 {
				result = append(result, []int{nums[i], nums[left], nums[right]})

				// Пропускаем дубликаты слева
				for left < right && nums[left] == nums[left+1] {
					left++
				}
				// Пропускаем дубликаты справа
				for left < right && nums[right] == nums[right-1] {
					right--
				}

				left++
				right--
			} else if sum < 0 {
				left++
			} else {
				right--
			}
		}
	}
	return result
}
```

**Идея:**

1. **Сортируем массив** (O(n log n))
2. **Фиксируем первый элемент** и превращаем задачу в **Two Sum II для отсортированного массива**
3. Для каждого первого элемента `nums[i]` найти две числа `nums[left]` и `nums[right]` такие что:
   - `nums[i] + nums[left] + nums[right] == 0`
   - `left = i + 1`, `right = len(nums) - 1`
   - Используем технику два указателя

4. **Пропускаем дубликаты** на всех уровнях чтобы избежать повторяющихся троек

## Пошаговый пример

**Входные данные:** `nums = [-1, 0, 1, 2, -1, -4]`

**Шаг 1: Сортируем**
```
[-4, -1, -1, 0, 1, 2]
```

**Шаг 2: Проходим по каждому элементу как "первый"**

| i | nums[i] | left | right | Сумма | Действие | Результат |
|---|---------|------|-------|-------|----------|-----------|
| 0 | -4 | 1 | 5 | -4+(-1)+2=-3 | < 0, left++ | - |
| 0 | -4 | 2 | 5 | -4+(-1)+2=-3 | < 0, left++ | - |
| 0 | -4 | 3 | 5 | -4+0+2=-2 | < 0, left++ | - |
| 0 | -4 | 4 | 5 | -4+1+2=-1 | < 0, left++ | - |
| 0 | -4 | 5 | 5 | out of bounds | - | - |
| 1 | -1 | 2 | 5 | -1+(-1)+2=0 | == 0, найдено! | [-1, -1, 2] |
| 1 | -1 | 3 | 4 | -1+0+1=0 | == 0, найдено! | [-1, 0, 1] |
| 1 | -1 | 4 | 3 | left >= right | stop | - |
| 2 | -1 | - | - | skip (дубликат) | - | - |
| 3 | 0 | 4 | 5 | 0+1+2=3 | > 0, right-- | - |
| 3 | 0 | 4 | 4 | left >= right | stop | - |
| 4 | 1 | - | - | stop (i < len-2) | - | - |

**Результат:** `[[-1, -1, 2], [-1, 0, 1]]` ✓

## Визуализация

```
Начальный массив: [-1, 0, 1, 2, -1, -4]

Шаг 1: Сортируем
[-4, -1, -1, 0, 1, 2]

Шаг 2: i = 0, nums[i] = -4
        Нужно найти пару (left, right) где nums[left] + nums[right] = 4
[-4, -1, -1, 0, 1, 2]
 ↑   ↑           ↑
 i  left       right
Сумма = -4 + (-1) + 2 = -3 < 0, двигаем left вправо

Шаг 3: i = 1, nums[i] = -1
        Нужно найти пару где nums[left] + nums[right] = 1
[-4, -1, -1, 0, 1, 2]
      ↑  ↑        ↑
      i left    right
Сумма = -1 + (-1) + 2 = 0 ✓ Найдено!

Продолжаем с left++, right--:
[-4, -1, -1, 0, 1, 2]
      ↑      ↑  ↑
      i    left right
Сумма = -1 + 0 + 1 = 0 ✓ Найдено!
```

## Ключевые оптимизации

### 1. Оптимизация: Проверка `if nums[i] > 0`

Если первый элемент положительный, то все остальные тоже положительны (массив отсортирован). Сумма трёх положительных чисел не может быть 0!

```go
if nums[i] > 0 {
    break  // Больше решений нет
}
```

### 2. Пропуск дубликатов первого элемента

Если мы уже нашли все тройки с `nums[i]`, не нужно проверять `nums[i+1]` если он равен `nums[i]`:

```go
if i > 0 && nums[i] == nums[i-1] {
    continue
}
```

### 3. Пропуск дубликатов слева и справа

После нахождения решения, пропускаем все дубликаты `left` и `right`:

```go
for left < right && nums[left] == nums[left+1] {
    left++
}
for left < right && nums[right] == nums[right-1] {
    right--
}
```

Это предотвращает добавление одинаковых тройк в результат.

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n²)** |
| **Space** | **O(1)** или **O(n)** |

**Разбор:**
- **Сортировка:** O(n log n)
- **Внешний цикл (for i):** O(n)
- **Внутренний цикл (Two Sum II):** O(n)
- **Всего:** O(n log n) + O(n²) = **O(n²)**

**Пространство:**
- **O(1):** если не считать выходной массив
- **O(n):** если считать пространство для сортировки

---

## Почему два указателя?

Если бы мы использовали hash set вместо двух указателей:

```go
func ThreeSumHashSet(nums []int) [][]int {
    seen := make(map[int]bool)
    var result [][]int
    
    for i := 0; i < len(nums)-2; i++ {
        for j := i + 1; j < len(nums); j++ {
            complement := -nums[i] - nums[j]
            if seen[complement] {
                result = append(result, []int{nums[i], nums[j], complement})
            }
            seen[nums[j]] = true
        }
    }
    return result
}
```

- **Time:** O(n²)
- **Space:** O(n)

Но два указателя **лучше**:
- Работает на отсортированном массиве
- Легче пропускать дубликаты
- Space O(1) (без внешних структур)

---

## Сравнение вариантов

| Подход | Time | Space | Преимущества |
|--------|------|-------|--------------|
| Два указателя + сортировка | O(n²) | O(1) | ✓ Оптимально, просто пропускать дубликаты |
| Hash Set | O(n²) | O(n) | Похоже, но больше памяти |
| Brute Force (3 цикла) | O(n³) | O(1) | ✗ Слишком медленно |

---

## Интересный факт

Эта задача демонстрирует, как **сортировка** может превратить O(n³) задачу (brute force с проверкой дубликатов) в O(n²) задачу (два указателя).

Это типичный паттерн в алгоритмике:
1. **Sort** → перестраивает структуру
2. **Two Pointers** → использует отсортированность
3. **Skipping duplicates** → обеспечивает уникальность
