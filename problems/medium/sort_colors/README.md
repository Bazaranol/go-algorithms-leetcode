# Sort Colors

## Условие

Дан массив `nums` содержащий значения **0**, **1** и **2**, представляющие три цвета (красный, белый, голубой). Нужно отсортировать массив **in-place** так, чтобы все нули шли первыми, потом единицы, потом двойки.

**Требования:**
- Использовать O(1) дополнительную память (не создавать новый массив)
- Модифицировать массив in-place
- Элементы могут быть только 0, 1 или 2
- Не использовать встроенную функцию сортировки

## Примеры

Input: `nums = [2, 0, 2, 1, 1, 0]`  
Output: `[0, 0, 1, 1, 2, 2]`  
Объяснение: Все нули, потом единицы, потом двойки

Input: `nums = [2, 0, 1]`  
Output: `[0, 1, 2]`  
Объяснение: Отсортировано

Input: `nums = [0]`  
Output: `[0]`  
Объяснение: Один элемент

Input: `nums = [2, 2, 0, 1, 2, 0]`  
Output: `[0, 0, 1, 2, 2, 2]`  
Объяснение: Все цвета разделены

---

## Подход: Алгоритм голландского национального флага (Dutch National Flag)

```go
func SortColors(nums []int) []int {
	slow := 0
	medium := 0
	fast := len(nums) - 1

	for medium <= fast {
		if nums[medium] == 0 {
			nums[slow], nums[medium] = nums[medium], nums[slow]
			medium++
			slow++
		} else if nums[medium] == 2 {
			nums[medium], nums[fast] = nums[fast], nums[medium]
			fast--
		} else {
			medium++
		}
	}

	return nums
}
```

**Идея:**

Используем **три указателя** для разделения массива на три региона:

1. **slow** — граница между 0s и 1s (позиция для следующего 0)
2. **medium** — текущий элемент, который проверяем
3. **fast** — граница между 1s и 2s (позиция для следующего 2)

**Три региона:**
- `[0, slow)` — все 0s
- `[slow, medium)` — все 1s
- `[medium, fast]` — необработанные элементы
- `(fast, n)` — все 2s

**Алгоритм:**

1. Если `nums[medium] == 0`:
   - Поменять со `slow` (переместить 0 в регион нулей)
   - `slow++`, `medium++`

2. Если `nums[medium] == 2`:
   - Поменять с `fast` (переместить 2 в регион двоек)
   - `fast--` (НЕ двигаем medium, потому что нужно проверить элемент с fast)

3. Если `nums[medium] == 1`:
   - Это на месте, просто `medium++`

## Пошаговый пример

**Входные данные:** `nums = [2, 0, 2, 1, 1, 0]`

| Шаг | slow | medium | fast | nums[medium] | Действие | Массив | Состояние |
|-----|------|--------|------|-------------|----------|--------|-----------|
| 0 | 0 | 0 | 5 | 2 | swap(0,5) | [0, 0, 2, 1, 1, 2] | fast-- |
| 1 | 0 | 0 | 4 | 0 | swap(0,0) | [0, 0, 2, 1, 1, 2] | slow++, medium++ |
| 2 | 1 | 1 | 4 | 0 | swap(1,0) | [0, 0, 2, 1, 1, 2] | slow++, medium++ |
| 3 | 2 | 2 | 4 | 2 | swap(2,4) | [0, 0, 1, 1, 2, 2] | fast-- |
| 4 | 2 | 2 | 3 | 1 | move | [0, 0, 1, 1, 2, 2] | medium++ |
| 5 | 2 | 3 | 3 | 1 | move | [0, 0, 1, 1, 2, 2] | medium++ |
| 6 | 2 | 4 | 3 | - | medium > fast, stop | [0, 0, 1, 1, 2, 2] | ✓ |

**Результат:** `[0, 0, 1, 1, 2, 2]` ✓

## Визуализация

```
Начальное состояние:
nums = [2, 0, 2, 1, 1, 0]
        ↑  ↑           ↑
      slow medium     fast

Обозначение регионов:
[0s]  [1s]  [необ.]  [2s]
----  ----  -------  ----

Итерация 1: nums[0] = 2
Swap с fast:
nums = [0, 0, 2, 1, 1, 2]
        ↑  ↑           ↑
      slow medium     fast
fast--

Итерация 2: nums[0] = 0
Swap со slow:
nums = [0, 0, 2, 1, 1, 2]
        ↑  ↑           ↑
      slow medium     fast
slow++, medium++

Итерация 3: nums[1] = 0
Swap со slow:
nums = [0, 0, 2, 1, 1, 2]
           ↑  ↑           ↑
         slow medium     fast
slow++, medium++

Итерация 4: nums[2] = 2
Swap с fast:
nums = [0, 0, 1, 1, 2, 2]
           ↑  ↑        ↑
         slow medium  fast
fast--

Итерация 5-6: nums[medium] = 1
Просто двигаем medium++

Финальный результат: [0, 0, 1, 1, 2, 2] ✓
```

## Ключевые моменты

### 1. Почему не двигаем medium когда nums[medium] == 2?

Когда мы меняем `nums[medium]` с `nums[fast]`, в позицию `medium` приходит неизвестный элемент. Нужно его проверить на следующей итерации!

```go
nums[medium], nums[fast] = nums[fast], nums[medium]
fast--
// НЕ двигаем medium! На следующей итерации проверим новый элемент
```

### 2. Почему двигаем оба (slow и medium) когда nums[medium] == 0?

Когда мы меняем `nums[medium]` с `nums[slow]`, мы знаем что:
- В позицию `medium` пришла **единица** (из региона 1s)
- Она уже на месте, поэтому двигаем `medium++`
- Увеличиваем `slow` для следующего нуля

### 3. Три указателя разделяют задачу

- **slow** управляет границей 0s/1s
- **fast** управляет границей 1s/2s
- **medium** проверяет текущий элемент и движется между ними

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |

## Историческая справка: Голландский национальный флаг

**Почему "Голландский национальный флаг"?**

Флаг Нидерландов состоит из трёх горизонтальных полос:
- Красная (сверху)
- Белая (в середине)
- Синяя (снизу)

Проблема была предложена **Эдсгером Дейкстрой** в 1976 году. Нужно было отсортировать массив с тремя значениями так, чтобы они образовали "флаг":
- 0 (красный) → левый край
- 1 (белый) → середина
- 2 (синий) → правый край

Алгоритм идеален для этой задачи!

## Альтернативные подходы

### 1. Counting Sort (Подсчёт)

```go
func SortColorsCountSort(nums []int) []int {
    count := [3]int{0, 0, 0}
    for _, num := range nums {
        count[num]++
    }
    
    idx := 0
    for color := 0; color < 3; color++ {
        for i := 0; i < count[color]; i++ {
            nums[idx] = color
            idx++
        }
    }
    return nums
}
```

**Сложность:** Time O(n), Space O(1) — тоже хорошо!

### 2. Встроенная сортировка

```go
func SortColorsBuiltin(nums []int) []int {
    sort.Ints(nums)
    return nums
}
```

**Сложность:** Time O(n log n), Space O(1) — медленнее!

## Сравнение подходов

| Подход | Time | Space | Примечание |
|--------|------|-------|-----------|

| Встроенная сортировка | O(n log n) | O(1) | ✗ Слишком медленно |

## Почему Dutch Flag лучше?

1. **Один проход:** Проходим массив один раз слева направо
2. **In-place:** Используем O(1) дополнительной памяти
3. **Простота:** Три указателя легко понять и реализовать
4. **Оптимальность:** Не можно быстрее чем O(n) для сортировки

## Интересный факт

Алгоритм **Dutch National Flag** работает для любого числа цветов, не только трёх! Например, для четырёх цветов можно использовать четыре указателя и т.д.

```
Для k цветов:
- Используем k указателей
- Каждый указатель управляет границей между двумя цветами
- Time O(n), Space O(1)
```
