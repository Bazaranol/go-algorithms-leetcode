# Container With Most Water

## Условие

Дан массив целых чисел `height`, где `height[i]` представляет **высоту** вертикальной линии в позиции `i`. Найти две линии, которые вместе с осью X образуют контейнер, вмещающий наибольшее количество воды.

**Требования:**
- Нельзя наклонять контейнер
- Объём воды = `min(height[i], height[j]) * (j - i)`
- Найти максимальный объём

## Примеры

Input: `height = [1, 8, 6, 2, 5, 4, 8, 3, 7]`  
Output: `49`  
Объяснение: Линии в позициях 1 и 8 с высотами 8 и 7  
Объём = `min(8, 7) * (8 - 1) = 7 * 7 = 49`

Input: `height = [1, 1]`  
Output: `1`  
Объяснение: Две линии с одинаковой высотой 1 на расстоянии 1  
Объём = `min(1, 1) * (1 - 0) = 1`

Input: `height = [2, 3, 4, 5, 18, 17, 6]`  
Output: `17`  
Объяснение: Линии 18 и 17 дают максимум: `min(18, 17) * (4 - 0) = 17 * 4 = 68`? Нет, давайте пересчитаем...  
На самом деле, линии 18 и 6: `min(18, 6) * (6 - 4) = 6 * 2 = 12`  
Линии 18 и 17: они рядом, расстояние = 1, объём = `min(18, 17) * 1 = 17` ✓

---

## Подход: Двухуказательная техника (Greedy Two-Pointer)

```go
func ContainerWithMostWater(height []int) int {
	left := 0
	right := len(height) - 1
	maxArea := 0

	for right > left {
		area := min(height[left], height[right]) * (right - left)
		maxArea = max(maxArea, area)

		if height[left] < height[right] {
			left++
		} else {
			right--
		}
	}

	return maxArea
}
```

**Идея:**

Используем **жадный подход** с двумя указателями:

1. **left** — указатель на начало (самая левая линия)
2. **right** — указатель на конец (самая правая линия)

**Алгоритм:**

1. Начинаем с максимальной ширины (от 0 до n-1)
2. Вычисляем площадь: `area = min(height[left], height[right]) * (right - left)`
3. Обновляем максимум
4. **Ключевой момент:** Двигаем указатель с меньшей высотой:
   - Если `height[left] < height[right]` → двигаем `left++`
   - Иначе → двигаем `right--`
5. Повторяем, пока `left < right`

---

## Почему двигаем указатель с меньшей высотой?

**Это критическая часть алгоритма!**

Обозначим высоты как `h[left]` и `h[right]`, где `h[left] < h[right]`.

Текущая площадь: `area = h[left] * width`

Если мы сдвинули `right` влево:
- Новая ширина: `width - 1` (уменьшилась)
- Новая высота: `min(h[left], h[right-1])`
  - Если `h[right-1] >= h[left]`, то площадь = `h[left] * (width - 1)` < `h[left] * width`
  - Если `h[right-1] < h[left]`, то площадь = `h[right-1] * (width - 1)` < `h[left] * width`
- **В обоих случаях площадь уменьшится!**

**Вывод:** Двигая более высокий указатель, мы не улучшим результат.  
Двигая **низкий указатель**, у нас есть **шанс** найти более высокую линию и потенциально увеличить площадь.

## Пошаговый пример

**Входные данные:** `height = [1, 8, 6, 2, 5, 4, 8, 3, 7]`

| Шаг | left | right | h[left] | h[right] | Ширина | Площадь | maxArea | Действие |
|-----|------|-------|---------|----------|--------|---------|---------|----------|
| 0 | 0 | 8 | 1 | 7 | 8 | 1*8=8 | 8 | h[0]<h[8], left++ |
| 1 | 1 | 8 | 8 | 7 | 7 | 7*7=49 | 49 | h[1]>h[8], right-- |
| 2 | 1 | 7 | 8 | 3 | 6 | 3*6=18 | 49 | h[1]>h[7], right-- |
| 3 | 1 | 6 | 8 | 8 | 5 | 8*5=40 | 49 | h[1]==h[6], right-- |
| 4 | 1 | 5 | 8 | 4 | 4 | 4*4=16 | 49 | h[1]>h[5], right-- |
| 5 | 1 | 4 | 8 | 5 | 3 | 5*3=15 | 49 | h[1]>h[4], right-- |
| 6 | 1 | 3 | 8 | 2 | 2 | 2*2=4 | 49 | h[1]>h[3], right-- |
| 7 | 1 | 2 | 8 | 6 | 1 | 6*1=6 | 49 | left >= right, stop |

**Результат:** maxArea = 49 ✓

## Ключевые идеи

1. **Жадный выбор:** Двигаем указатель с меньшей высотой, потому что это единственный способ потенциально улучшить результат.

2. **Доказуемость:** Если мы пропустим некоторую пару (i, j) из-за жадного выбора, она не может дать лучший результат, чем текущий.

3. **Оптимальность:** За O(n) пройдём все "интересные" пары, не проверяя все O(n²) возможностей.

---

## Сложность

| Метрика | Значение |
|---------|----------|
| **Time** | **O(n)** |
| **Space** | **O(1)** |

- **Time O(n)**: Каждый указатель проходит массив один раз, всего 2n операций
- **Space O(1)**: Используем только два указателя, без дополнительных структур

---

## Почему не O(n²)?

**Наивный подход (brute force):**

```go
func ContainerWithMostWaterBrute(height []int) int {
    maxArea := 0
    for i := 0; i < len(height); i++ {
        for j := i + 1; j < len(height); j++ {
            area := min(height[i], height[j]) * (j - i)
            maxArea = max(maxArea, area)
        }
    }
    return maxArea
}
```

**Сложность:** O(n²) — проверяем все пары

**Мой подход (two-pointer):**

- Не проверяем все пары
- Пропускаем пары, которые гарантированно не будут оптимальными
- **O(n)** — намного лучше!

---

## Интересный факт

Эта задача показывает, как **правильный выбор стратегии** может превратить O(n²) задачу в O(n):

- **Brute Force:** Проверить всё → O(n²)
- **Greedy Two-Pointer:** Быть умнее при выборе → O(n)

Алгоритм работает потому что **ширина только уменьшается**, а **высота может только увеличиться** (когда мы двигаем низкий указатель). Это естественно ведёт нас к оптимуму.

---

## Сравнение подходов

| Подход | Time | Space | Примечание |
|--------|------|-------|-----------|
| Brute Force | O(n²) | O(1) | ✗ Слишком медленно |
| Two Pointer | O(n) | O(1) | ✓ **Оптимально!** |
